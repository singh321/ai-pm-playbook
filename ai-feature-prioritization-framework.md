# AI Feature Prioritization Framework

A practical framework for deciding which AI features to build first.

---

## Why AI prioritization is harder

Prioritizing AI features is harder than normal feature prioritization because AI adds:
- uncertainty in output quality
- trust risk
- cost and latency trade-offs
- workflow complexity
- unclear user adoption patterns

That means not every exciting AI idea should be built first.

---

## My prioritization framework

I evaluate AI feature ideas on 5 dimensions:

1. User pain
2. Workflow fit
3. AI advantage
4. Feasibility and control
5. Business impact

---

## 1. User pain

### Questions
- Is this a real pain point?
- How frequent is the pain?
- How costly is it today?
- Is it a top workflow problem or a nice-to-have?

High priority features usually solve frequent and painful tasks.

---

## 2. Workflow fit

### Questions
- Does this fit naturally into an existing workflow?
- Will users use it in the flow of work?
- Does it reduce steps or create more steps?
- Is the moment of value obvious?

A powerful AI feature that does not fit workflow often fails.

---

## 3. AI advantage

### Questions
- Is AI meaningfully better than rule-based logic here?
- Does the use case benefit from summarization, generation, extraction, ranking, or reasoning?
- Is AI solving something that otherwise takes high manual effort?

If AI is not creating meaningful leverage, it should not be the default solution.

---

## 4. Feasibility and control

### Questions
- Can we measure quality?
- Can we control risk?
- Can we start with human-in-the-loop if needed?
- Is latency acceptable?
- Are data and infrastructure ready?

Early AI wins often come from use cases where quality can be evaluated and risk can be controlled.

---

## 5. Business impact

### Questions
- Will this help activation, retention, expansion, or revenue?
- Does it make the product more differentiated?
- Can it influence paid adoption?
- Does it reduce internal cost?

High-priority features create both user and business value.

---

## Scoring model

I score each feature from 1–5 across the 5 dimensions:

- user pain
- workflow fit
- AI advantage
- feasibility and control
- business impact

Then I look at:
- total score
- risk profile
- sequencing logic
- dependencies

---

## What usually wins first

The best first AI features often:
- solve a repeated workflow pain
- save time
- fit naturally into the product
- are easy to evaluate
- start with assistive value
- build trust early

Examples:
- summarization
- enrichment
- classification
- content drafting
- ranking / prioritization
- workflow copilots

---

## What I avoid early

I avoid starting with:
- vague “chat for everything” features
- high-risk fully autonomous workflows
- hard-to-evaluate reasoning tasks
- use cases with weak user pain
- AI ideas without monetization or retention upside

---

## Final principle

The right AI feature to build first is not the most impressive one.

It is the one that:
- solves clear pain
- fits workflow
- creates trusted value
- can be measured
- supports business outcomes
